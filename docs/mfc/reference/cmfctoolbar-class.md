# <a name="cmfctoolbar-class"></a>CMFCToolBar (classe)
Le `CMFCToolBar` ressemble à [classe CToolBar](../../mfc/reference/ctoolbar-class.md), mais fournit la prise en charge supplémentaire pour les fonctionnalités d’interface utilisateur. Celles-ci incluent les barres d'outils à deux dimensions, les barres d'outils avec images interactives, les grandes icônes, les boutons pager, les barres d'outils verrouillées, les contrôles rebar, le texte sous les images, les images d'arrière-plan et les barres d'outils avec onglets. La classe `CMFCToolBar` contient également une prise en charge intégrée de la personnalisation des barres d'outils et des menus par l'utilisateur, du glisser-déplacer entre les barres d'outils et les menus, des boutons de zone de liste modifiable, des boutons de zone d'édition, des sélecteurs de couleurs et des boutons de regroupement.  
  
## <a name="syntax"></a>Syntaxe  
  
```  
class CMFCToolBar : public CMFCBaseToolBar  
```

## <a name="remarks"></a>Notes  
 Pour incorporer un `CMFCToolBar` de l’objet dans votre application, procédez comme suit :  
  
1.  Ajouter un `CMFCToolBar` objet à la fenêtre frame principale.  
  
2.  Lorsque vous traitez le `WM_CREATE` de messages pour la fenêtre frame principale, appelez [CMFCToolBar::Create](#create) ou [CMFCToolBar::CreateEx](#createex) pour créer la barre d’outils et de spécifier son style.  
  
3.  Appelez [CBasePane::EnableDocking](../../mfc/reference/cbasepane-class.md#enabledocking) pour spécifier le style d’ancrage.  
  
 Pour insérer un bouton spécial, comme une zone de liste déroulante ou de la barre d’outils de la liste déroulante, réserver un bouton factice dans la ressource parente et remplacez le bouton fictif lors de l’exécution à l’aide de [CMFCToolBar::ReplaceButton](#replacebutton). Pour plus d’informations, consultez [procédure pas à pas : placer le contrôle sur la barre de d’outils](../walkthrough-putting-controls-on-toolbars.md).  
  
 `CMFCToolBar`est la classe de base pour les classes de la bibliothèque MFC [CMFCMenuBar classe](../../mfc/reference/cmfcmenubar-class.md), [CMFCPopupMenuBar classe](../../mfc/reference/cmfcpopupmenubar-class.md), et [CMFCDropDownToolBar classe](../../mfc/reference/cmfcdropdowntoolbar-class.md).

 Le framework appelle cette méthode lorsque l’utilisateur clique sur le **afficher les boutons sur deux lignes** bouton.  
  
 Appelez le [CMFCToolBar::SetSiblingToolBar](#setsiblingtoolbar) méthode pour permettre la **afficher les boutons sur une ligne** ou **afficher les boutons sur deux lignes** boutons. Si vous appelez [CMFCToolBar::SetSiblingToolBar](#setsiblingtoolbar) pour cette barre d’outils, la barre d’outils frères est déplacé vers une ligne distincte. Dans le cas contraire, cette barre d’outils est déplacée vers une ligne distincte.  
  
 Le framework appelle la [CMFCToolBar::SetOneRowWithSibling](#setonerowwithsibling) méthode lorsque l’utilisateur clique sur le **afficher les boutons sur une ligne** bouton.  
  
##  <a name="setuserimages"></a>CMFCToolBar::SetUserImages  
 Définit la collection d’images défini par l’utilisateur de l’application.  
  
```  
static BOOL SetUserImages(CMFCToolBarImages* pUserImages);
```  
  
### <a name="parameters"></a>Paramètres  
 [in] `pUserImages`  
 Pointeur vers la collection d’images défini par l’utilisateur.  
  
### <a name="return-value"></a>Valeur de retour  
 Différent de zéro si la méthode réussit ; Sinon, 0 si spécifié `CMFCToolBarImages` objet n’est pas valide ou a une taille d’image qui diffère de la taille de l’image par défaut de la barre d’outils.  
  
### <a name="remarks"></a>Remarques  
 L’infrastructure utilise des images définies par l’utilisateur à dessiner des boutons de barre d’outils qui sont personnalisées par l’utilisateur. La liste d’images spécifiée par `pUserImages` est partagé entre toutes les barres d’outils dans l’application.  
  
 Cette méthode génère un échec d’assertion dans les versions Debug si spécifié `CMFCToolBarImages` objet n’est pas valide ou a une taille d’image qui diffère de la taille de l’image par défaut de la barre d’outils.  
  
 Les exemples OutlookDemo ToolTipDemo et VisualStudioDemo utilisent cette méthode pour définir la collection globale des images définies par l’utilisateur. Ils chargent le fichier nommé UserImages.bmp, qui se trouve dans le répertoire de travail de l’application.  
  
 Appelez le [CMFCToolBar::GetUserImages](#getuserimages) méthode pour récupérer la collection d’images de défini par l’utilisateur dans l’application.

virtual CSize StretchPane(
    int nLength,

### <a name="return-value"></a>Valeur de retour

### <a name="remarks"></a>Notes

 La valeur de retour est déterminée en appelant [CMFCToolBar::CalcSize](#calcsize).

##  <a name="wraptoolbar"></a>CMFCToolBar::WrapToolBar  
 Repositionne les boutons de barre d’outils dans les dimensions spécifiées.  
  
```  
int WrapToolBar(
    int nWidth,  
    int nHeight = 32767,  
    CDC* pDC = NULL,  
    int nColumnWidth = -1,  
    int nRowHeight = -1);
```  
  
### <a name="parameters"></a>Paramètres  
 [in] `nWidth`  
 Largeur maximale de la barre d’outils.  
  
 [in] `nHeight`  
 Hauteur maximale de la barre d’outils. Pas utilisé si la barre d’outils est flottante.  
  
 [in] `pDC`  
 Pointeur vers un contexte de périphérique. Si NULL, le contexte de périphérique pour la barre d’outils est utilisé.  
  
 [in] `nColumnWidth`  
 Largeur du bouton. Si-1, la largeur actuelle est utilisée.  
  
 [in] m`nRowHeight`  
 Hauteur du bouton. Si-1, la hauteur actuelle est utilisée.  
  
### <a name="return-value"></a>Valeur de retour  
 Le nombre de lignes de boutons de la barre d’outils.  
  
### <a name="remarks"></a>Remarques  
 Cette méthode repositionne les boutons dans la barre d’outils, en encapsulant des boutons pour les lignes supplémentaires si nécessaire.
